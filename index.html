GitHub Actions — Guia Rápido

Workflows automatizados para CI/CD no GitHub

O que são?

GitHub Actions é a plataforma de automação do GitHub. Você cria workflows (arquivos YAML) que rodam em resposta a eventos como push, pull_request ou no agendamento (cron).

Conceitos

  • Workflow: arquivo YAML em .github/workflows/.
  • Jobs: grupos de passos executados em um runner.
  • Steps: comandos ou actions individuais.
  • Runners: máquinas que executam os jobs (ex.: ubuntu-latest).
  • Events: gatilhos (push, PR, release, schedule... ).

Exemplo de workflow

# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [ "main" ]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
      - name: Instalar deps e testar
        run: |
          npm ci
          npm test --if-present

Crie a pasta .github/workflows e salve como ci.yml.
